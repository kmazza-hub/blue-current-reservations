# Blue Current V32.3 — Core Cleanup

## Release objective
Establish a stable baseline before Scheduling, Payroll, Purchasing, and new integrations.

## Changes
- Retired the mismatched legacy Autonomous Operations browser module that caused the final null-DOM startup error.
- Preserved retired code under `client/js/legacy/` for reference without loading it.
- Added lifecycle-aware module states: `ready`, `skipped`, `blocked`, and `failed`.
- Added deterministic startup reports with module counts, dependency status, and elapsed startup time.
- Loaded previously omitted Executive Command Center and Inventory Intelligence client modules.
- Removed the retired autonomous dependency from Guest Intelligence.
- Updated startup diagnostics to report the actual V32.3 build and tolerate absent diagnostic DOM.
- Rotated the browser authentication storage key for a clean V32.3 session.
- Expanded validation to detect duplicate script loads, duplicate HTML IDs, active legacy scripts, and version mismatch.

## Expected console result
`[Blue Current V32.3.0] startup complete in ...ms`

The retired modules appear as `skipped`, not as runtime errors.
