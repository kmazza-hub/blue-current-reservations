# Blue Current V32.4 — Platform Foundation

## Purpose
V32.4 establishes a reusable runtime foundation before customer-facing scheduling work begins.

## Delivered
- Central `BlueCurrentPlatform` runtime.
- Deterministic `ModuleRegistry` with duplicate protection, dependency blocking, timings, optional/skipped states, and immutable reports.
- Global error boundary for browser errors and unhandled promise rejections.
- Runtime diagnostics API available at `window.BlueCurrentPlatformRuntime.diagnostics()`.
- Single startup completion event through `registry.complete()`.
- Safer initial DOM access for shared clock and copyright rendering.
- Cache/version rotation to 32.4.0.

## Browser diagnostics
Open DevTools and run:

```js
window.BlueCurrentPlatformRuntime.diagnostics()
```

The result contains build information, module lifecycle status, startup duration, and captured runtime errors.

## Next release
V33 Scheduling should register through the V32.4 platform registry and expose the standard `init`, `load`, `render`, and `destroy` lifecycle where applicable.
